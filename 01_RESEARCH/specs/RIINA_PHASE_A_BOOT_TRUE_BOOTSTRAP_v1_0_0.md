### ANNOTATION REQUIREMENTS

For EVERY hex byte or group, provide:
1. What it represents (field name)
2. What the value means
3. Why this value was chosen
4. Reference to specification or standard (where applicable)

For EVERY instruction, provide:
1. The assembly mnemonic
2. What the instruction does
3. How it fits into the algorithm
4. The exact encoding breakdown

### STRUCTURE

The file must have these sections (clearly marked):

1. FILE HEADER (metadata, verification info, build instructions)
2. ELF HEADER (e_ident, e_type, e_machine, etc.)
3. PROGRAM HEADER (p_type, p_flags, p_offset, etc.)
4. CODE: ENTRY POINT (initialization)
5. CODE: MAIN LOOP (read, parse, output)
6. CODE: HEX PARSING (convert char to nibble)
7. CODE: ERROR HANDLING (exit with error)
8. CODE: SUCCESS EXIT (exit with 0)

### VERIFICATION AIDS

Include in comments:
- Running byte offset for every section
- Expected binary size after each section
- Cross-references between related code sections
- Alternative implementations considered and rejected

## OUTPUT FORMAT

The output IS the hex file itself. No markdown wrapper.

After the hex file, provide:
1. Expected SHA-256 of the hex file
2. Expected SHA-256 of the binary
3. Expected binary size
4. Disassembly of the code section

## 11.2 GitHub Actions Workflow

```yaml
# .github/workflows/bootstrap.yml
name: Bootstrap Chain

on:
  push:
    paths:
      - 'bootstrap/**'
  pull_request:
    paths:
      - 'bootstrap/**'
  workflow_dispatch:  # Manual trigger

jobs:
  stage-0:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Build hex seed
        run: |
          cd bootstrap/stage-0-hex
          grep -v '^#' teras-hex-seed.hex | xxd -r -p > seed
          chmod +x seed
          
      - name: Verify hash
        run: |
          cd bootstrap/stage-0-hex
          sha256sum seed | tee computed.sha256
          diff computed.sha256 seed.sha256
          
      - name: Run tests
        run: |
          cd bootstrap/stage-0-hex/tests
          ./run_all_tests.sh
          
      - name: Upload artifact
        uses: actions/upload-artifact@v4
        with:
          name: stage-0-seed
          path: bootstrap/stage-0-hex/seed

  stage-1:
    needs: stage-0
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      
      - name: Download seed
        uses: actions/download-artifact@v4
        with:
          name: stage-0-seed
          path: build/
          
      - name: Build assembler
        run: |
          chmod +x build/seed
          ./build/seed < bootstrap/stage-1-asm/teras-asm.hex > build/teras-asm
          chmod +x build/teras-asm
          
      # ... continues for all stages

  verify-chain:
    needs: [stage-0, stage-1, stage-2, stage-3, stage-4]
    runs-on: ubuntu-latest
    steps:
      - name: Verify self-hosting
        run: |
          # terasc-v2 and terasc-v3 must be identical
          diff build/terasc-v2 build/terasc-v3
          
      - name: Verify reproducibility
        run: |
          # Rebuild entire chain, compare hashes
          ./bootstrap/verify-chain.sh
```
